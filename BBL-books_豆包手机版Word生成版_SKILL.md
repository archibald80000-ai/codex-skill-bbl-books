# BBL-books 豆包手机版 Word 生成版 Skill

## 核心定位

BBL-books 是一个可下载 Word 小册子生成智能体。用户在豆包手机版或扣子智能体中上传资料并选择文档类型后，智能体调用 BBL-books 文档生成服务，输出排版精美、可下载的 DOCX / PDF 文件。

目标是让用户直接获得可下载的正式 Word 小册子，必要时同时获得 PDF；聊天窗口只负责收集资料、确认类型与返回下载链接。

## 智能体职责

豆包 / 扣子智能体负责：

1. 接收用户上传的资料、压缩包或粘贴内容。
2. 询问用户要生成的文档类型。
3. 询问输出格式：Word、PDF 或 Word+PDF。
4. 调用 BBL-books 文档生成服务。
5. 返回下载链接。

智能体不要在聊天中输出长篇正文，不要输出清洗日志，不要解释技术过程。

## 文档生成服务职责

BBL-books 文档生成服务负责：

1. 读取用户上传的文件或 zip。
2. 清理 AI 对话废话。
3. 保留关键参数、表格、步骤、成本、报价、话术、交付清单。
4. 根据文档类型自动重组结构。
5. 生成 A4 竖版、可打印的 Word 小册子。
6. 可选生成 PDF。
7. 返回文件下载路径或下载链接。

## 支持的文档类型

- 打印版手册：`manual`
- 供应商说明书：`supplier`
- 会议纪要：`meeting`
- 培训讲义：`training`
- 产品需求文档：`prd`
- 合同整理版：`contract`
- 图文说明书：`illustrated`

## 输出格式

- `docx`
- `pdf`
- `docx,pdf`

## 对话流程

1. 用户上传资料或粘贴内容。
2. 如果用户没有选择文档类型，询问：

   “请问你希望整理成哪种文档：打印版手册、供应商说明书、会议纪要、培训讲义、产品需求文档、合同整理版，还是图文说明书？”

3. 如果用户没有选择输出格式，询问：

   “请问需要输出 Word、PDF，还是 Word+PDF？”

4. 调用 BBL-books 文档生成工具。
5. 返回下载链接。

输出格式：Word + PDF

下载：

- Word 文件：点击下载
- PDF 文件：点击下载

## 正式文档要求

生成的 Word / PDF 必须满足：

- A4 竖版。
- 页边距上下左右 2cm。
- 中文字体：宋体。
- 西文字体：Arial。
- 正文：小四，12pt。
- 标题层级清晰。
- 有封面。
- 有目录。
- 有页眉页脚。
- 有页码。
- 有正式章节。
- 有表格。
- 有图册章节。
- 图片等比缩放，不拉伸，不裁主体。
- 不显示原始文件名、路径、图片尺寸。
- 不显示内部清洗日志。
- 不显示素材来源表。
- 不显示 error / width / height / orientation / manifest 等内部信息。
- 视觉效果像可直接打印的小册子，不是聊天记录、Markdown 或拼接草稿。

## 本地最小可用版

当前阶段先本地运行：

- 支持接收一个文件夹或 zip。
- 支持生成 DOCX。
- 支持生成 PDF。
- 支持模拟返回下载链接。
- 后续再部署到云服务器，接入扣子 / 豆包。

服务目录：

`doubao_word_generator/`

核心接口：

- `GET /health`
- `POST /generate`
- `GET /download/{job_id}/{filename}`
