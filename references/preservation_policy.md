# Preservation Policy

BBL-books is not a summarizer. Preserve source detail and build a book-like manual around it.

## Must Preserve

- Implementation details and exact operational steps.
- Parameters, ratios, colors, model numbers, category numbers, route names, file paths, and identifiers.
- Dialogue scripts, sales scripts, teaching scripts, prompts, and user-facing wording.
- Tables and spreadsheet-derived rows.
- Pricing, quotations, cost calculations, budget assumptions, and formulas.
- Version information, applicable scenarios, legal/compliance notes, cautions, and delivery requirements.

## Conservative Dedupe

Only merge exact or near-exact duplicated filler. Do not delete repeated core parameters. If several files repeat important details, keep a single clean occurrence in the main chapter and list meaningful differences in notes or appendix.

## Reorganization

Reorder content into chapters only when it improves readability. Do not compress away the source's working detail. Prefer headings, lists, and tables over narrative paraphrase.

## Canonical Manual Merge

Do not concatenate files by filename. Classify paragraphs into formal topics such as compliance, design principles, body shape, facial design, identity marks, colors, personality/actions, hardware patent, registration process, delivery checklist, execution checklist, and image album. Keep each core parameter once in the main chapter; put version differences and source filenames in appendices.

For the first stable version, do not put version-difference analysis into the formal document. If differences must be tracked, write them only to the internal report.
